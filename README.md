# "Student Attendance System"


## Project Description

This project is a simple Student Attendance System developed collaboratively using GitHub. The project demonstrates the use of repositories, branches, commits, pull requests, and merge operations. 

---

## GitHub Workflow

- 1.The project repository was created by Seyedeh Sedigheh Hasheminezhad on GitHub.
- 2.Initial project structure was created, including a base README file in the main branch.
- 3.Team members were added as collaborators to the repository.
- 4.Each team member cloned the repository from GitHub and set up their local working environment.
- 5.Each member created their own feature branch from the main branch.
- 6.Each member worked independently on their assigned module in their respective branches.
- 7.The README.md file was shared across all branches, and each member updated the same file by adding their own contribution section.
- 8.Each member committed their changes locally with meaningful commit messages.
- 9.All changes were pushed to GitHub under their respective feature branches.
- 10.Each team member created a Pull Request (PR) from their feature branch to the main branch.
- 11.Pull Requests were reviewed to ensure correctness and consistency of the project structure.
- 12.All Pull Requests were merged into the main branch by Seyedeh Sedigheh Hasheminezhad after review.
- 13.After merging all branches, the final main branch contains a single unified README.md file including:
    - Project description
    - GitHub workflow documentation
    - Individual contributions of all team members
- 14.The final repository represents a fully collaborative GitHub workflow project demonstrating branching, committing, pull requests, and merging.
